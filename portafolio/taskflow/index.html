Qué es

Un backend como los que usan los productos serios

TaskFlow organiza proyectos, tareas y equipos — al estilo de herramientas como Trello o Linear — y demuestra cómo construimos sistemas a la medida: cada pieza validada, probada y lista para producción.

  • Registro e inicio de sesión con JWT y rotación de refresh tokens
  • Proyectos y tareas con filtros, búsqueda y paginación
  • Comentarios y roles de equipo (administrador / miembro)
  • Validación de datos en cada endpoint con Zod
  • Límite de peticiones y headers de seguridad en producción
  • Documentación interactiva OpenAPI / Swagger

        Cómo está construida
        
Arquitectura en tres capas

        
La lógica no vive amontonada en el endpoint: cada capa tiene una sola responsabilidad, se prueba por separado y se puede cambiar sin romper el resto.

      


      

        

          01
          
Controller

          
Recibe la petición HTTP, extrae los datos ya validados y regresa la respuesta. Cero lógica de negocio.

          tasks.controller.ts
        

        

          02
          
Service

          
Aplica las reglas del negocio: ¿este usuario es miembro del proyecto? ¿puede borrar este comentario?

          tasks.service.ts
        

        

          03
          
Repository

          
Habla con la base de datos a través de Prisma, con consultas tipadas de extremo a extremo.

          tasks.repository.ts
